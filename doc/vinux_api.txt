te#utils#confirm({str}, {menu_list}, {action})            *te#utils#confirm*
    Display a confirmation dialog with customizable options.
    Automatically adapts to the editor environment:
    - Neovim (>=0.5): Uses floating window with custom keymaps
    - Vim 8+: Uses popup menu with filter support
    - Older Vim: Falls back to built-in confirm() dialog

    Parameters:
    - {str}: The prompt/message to display
    - {menu_list}: List of option strings to present
    - {action}: Callback when option is selected. Can be:
        - A function reference: executed with selected index (1-based)
        - A string command name: executed directly
        - A list [func, arg]: calls call(func, [arg, selected_index])
        - A dict {func: ..., arg: [...]}: calls call(func, extend(arg, [selected_index]))
        - Empty list []: no callback, returns value directly (classic mode)

    Returns:
    - Neovim/Vim8 async mode: Returns 0 immediately
    - Classic confirm mode: Returns the selected menu item string

    Key Bindings (Neovim floating window):
    <CR>   - Confirm current selection
    <C-c>  - Cancel/close without selection
    y      - Select first 'y' option (case-insensitive)
    n      - Select first 'n' option (case-insensitive)

te#utils#GetError({command}, {err_str})                    *te#utils#GetError*
    Test if a vim command produces an error.
    Parameters:
    - {command}: Vim command (not shell command) to test
    - {err_str}: Error substring pattern that is expected
    Returns: 0 if no error exists, -1 if error matches err_str

te#utils#EchoWarning({str}, [{level}])                   *te#utils#EchoWarning*
    Echo a warning/info message with auto-dismiss.
    Parameters:

    - {level}: Optional - 'err', 'warn' (default), or 'info'

    Message is displayed in a floating/popup window and auto-closes
    after the duration specified by g:message_delay_time.cur_val.

    Create a new project with session, compile flags, coding style and
    cscope info. This function guides the user through project creation:
    1. Prompts for project name

    3. Copies YCM config if completion is enabled
